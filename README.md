# МОНОПОЛИЯ · «Биржа 1929» — онлайн-игра

Локальная онлайн-Монополия с пародийными корпорациями. Node 22, **без npm-зависимостей**.

## Запуск

```bash
node monopoly/server.js          # http://localhost:3200
node monopoly/server.js --port 4000
```

## Что умеет

- **Аккаунты**: регистрация с подтверждением почты (демо-почта «📮 Почтовое отделение» —
  письма не покидают сервер, код виден в модалке и в консоли сервера), вход, пародийный
  «Goggle»-вход. Пароли — scrypt, сессии — httpOnly cookie. Хранение: `data/users.json`.
- **Комнаты**: создание (2–6 игроков, капитал, таймер хода, приватность), вход по коду,
  выбор фишки, **боты** (Botschild, Monopolina…), чат.
- **Полный движок Монополии** (`engine.js`, чистый модуль): рента с группами и ×2, дома/
  штаб-квартиры (even-rule), залог/выкуп, аукционы, сделки между игроками, тюрьма
  («под следствием»), карты Биржа/Совет, долги и банкротство, победа последнего выжившего.
- **Реалтайм**: SSE-поток (`/api/stream`), полные снапшоты состояния + события для анимаций.
  Переподключение автоматом; закрыл вкладку — за тебя доигрывает таймер.
- **RU/EN** на лету, выбор сохраняется.

## Структура

```
monopoly/
├── server.js        # HTTP + статика + auth + комнаты + SSE + таймеры + боты
├── engine.js        # чистый игровой движок (правила, доска, карты)
├── SPEC.md          # спецификация протокола и правил (контракт всех частей)
├── test/            # engine.test.js (node --test), sim.js (бот-партии)
├── data/            # users.json (создаётся сам)
├── public/          # клиент: index.html, app.js (оболочка), game.js (игра),
│                    #   styles.css, game.css, i18n.js, logos.js
└── design/          # исходный дизайн-мокап (архив)
```

## Тесты

```bash
node --test monopoly/test/engine.test.js
node monopoly/test/sim.js        # монте-карло бот-партий
```

## Ограничения демо

- Письма не отправляются наружу (нет SMTP) — демо-ящик. Точка расширения: `sendMail()` в server.js.
- «Goggle» — пародийный OAuth (реальный Google OAuth требует client id — можно добавить позже).
- Партии живут в памяти процесса (аккаунты — на диске).
